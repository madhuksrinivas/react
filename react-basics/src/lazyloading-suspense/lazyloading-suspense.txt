LAZY LOADING & SUSPENSE IN REACT
==================================

Lazy loading is a technique to defer loading of non-critical resources at page load time. 
React.lazy() and Suspense enable you to split your code and load components on demand, 
improving initial load performance.

1. WHAT IS LAZY LOADING?
------------------------
Lazy loading is a design pattern that delays loading of code until it's actually needed.
Instead of loading the entire application at once, you load only what's necessary for 
the initial render, and load additional code as users navigate or interact.

Benefits:
- Faster initial page load
- Reduced initial bundle size
- Better performance on slow networks
- Improved user experience
- Lower bandwidth consumption

2. REACT.LAZY()
---------------
React.lazy() is a function that lets you render a dynamic import as a regular component.
It automatically loads the component bundle when it's first rendered.

Syntax:
-------
const MyComponent = lazy(() => import('./MyComponent'));

How it works:
- Takes a function that calls a dynamic import()
- Returns a Promise that resolves to a module with a default export
- The component is loaded and cached when first rendered
- Subsequent renders use the cached version

Example:
--------
import { lazy } from 'react';

// Without lazy loading - loaded immediately
import Dashboard from './Dashboard';

// With lazy loading - loaded when rendered
const Dashboard = lazy(() => import('./Dashboard'));

3. SUSPENSE COMPONENT
---------------------
Suspense is a component that wraps lazy-loaded components and displays a fallback 
UI while waiting for the component to load.

Syntax:
-------
<Suspense fallback={<LoadingSpinner />}>
  <LazyComponent />
</Suspense>

Props:
- fallback: React element to show while waiting (required)
- children: The lazy-loaded components

Key Features:
- Shows fallback UI during loading
- Can wrap multiple lazy components
- Can be nested for granular loading states
- Works with React.lazy() and concurrent features

Example:
--------
<Suspense fallback={<div>Loading...</div>}>
  <LazyComponent />
</Suspense>

4. CODE SPLITTING BENEFITS
---------------------------
Code splitting divides your bundle into smaller chunks that can be loaded on demand.

Performance Benefits:
1. Reduced Initial Bundle Size
   - Users download less code upfront
   - Faster Time to Interactive (TTI)
   
2. Improved Load Time
   - Critical path is smaller
   - Faster First Contentful Paint (FCP)
   
3. Better Caching
   - Unchanged chunks remain cached
   - Only modified code needs re-download
   
4. Optimized Resource Usage
   - Load code only when needed
   - Reduced memory footprint

Example Bundle Sizes:
Without splitting: 500KB initial bundle
With splitting: 150KB initial + 100KB per route

5. WHEN TO USE LAZY LOADING
----------------------------
Ideal Use Cases:

✅ Route-based Splitting
   - Different pages/routes
   - User navigates between sections
   
✅ Modal/Dialog Components
   - Shown on user action
   - Not always visible
   
✅ Tab/Accordion Content
   - Multiple content sections
   - Only one visible at a time
   
✅ Heavy Components
   - Large data visualizations
   - Rich text editors
   - Video players
   
✅ Feature Flags
   - Conditional features
   - A/B testing variants
   
✅ User Role-based Content
   - Admin panels
   - Premium features

❌ Avoid for:
   - Small components (overhead > benefit)
   - Critical above-the-fold content
   - Components always visible on initial load

6. BEST PRACTICES
-----------------
1. Strategic Splitting
   - Split at route level first
   - Then split heavy components
   - Measure impact with metrics
   
2. Meaningful Fallbacks
   - Show skeleton screens
   - Use loading spinners
   - Maintain layout stability
   - Avoid layout shift
   
3. Error Boundaries
   - Wrap Suspense with ErrorBoundary
   - Handle loading failures gracefully
   - Provide retry mechanisms
   
4. Preloading
   - Preload on hover for better UX
   - Use route prefetching
   - Cache loaded components
   
5. Bundle Analysis
   - Use webpack-bundle-analyzer
   - Monitor chunk sizes
   - Optimize imports

Example with Error Boundary:
-----------------------------
<ErrorBoundary fallback={<ErrorMessage />}>
  <Suspense fallback={<Loading />}>
    <LazyComponent />
  </Suspense>
</ErrorBoundary>

7. COMMON PATTERNS
------------------
Pattern 1: Route-based Splitting
---------------------------------
import { lazy, Suspense } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const Dashboard = lazy(() => import('./pages/Dashboard'));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<PageLoader />}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/dashboard" element={<Dashboard />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}

Pattern 2: Component-level Splitting
-------------------------------------
const HeavyChart = lazy(() => import('./HeavyChart'));

function Dashboard() {
  const [showChart, setShowChart] = useState(false);
  
  return (
    <div>
      <button onClick={() => setShowChart(true)}>
        Show Chart
      </button>
      {showChart && (
        <Suspense fallback={<ChartSkeleton />}>
          <HeavyChart />
        </Suspense>
      )}
    </div>
  );
}

Pattern 3: Named Exports (wrap in object)
------------------------------------------
// If component uses named export, wrap it
const MyComponent = lazy(() => 
  import('./MyComponent').then(module => ({
    default: module.MyNamedComponent
  }))
);

Pattern 4: Preloading on Hover
-------------------------------
const Dashboard = lazy(() => import('./Dashboard'));

function NavLink() {
  const handleMouseEnter = () => {
    // Preload component before navigation
    import('./Dashboard');
  };
  
  return (
    <Link 
      to="/dashboard" 
      onMouseEnter={handleMouseEnter}
    >
      Dashboard
    </Link>
  );
}

Pattern 5: Multiple Components in Suspense
-------------------------------------------
<Suspense fallback={<Loading />}>
  <LazyComponent1 />
  <LazyComponent2 />
  <LazyComponent3 />
</Suspense>
// All components share the same loading state

Pattern 6: Nested Suspense (Granular Loading)
----------------------------------------------
<Suspense fallback={<PageLoader />}>
  <Header />
  <Suspense fallback={<ContentLoader />}>
    <Content />
  </Suspense>
  <Suspense fallback={<SidebarLoader />}>
    <Sidebar />
  </Suspense>
</Suspense>

8. ERROR HANDLING WITH LAZY LOADING
------------------------------------
Lazy loading can fail due to network issues or missing chunks. Always handle errors.

Using Error Boundaries:
-----------------------
class LazyLoadErrorBoundary extends Component {
  state = { hasError: false };
  
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  
  render() {
    if (this.state.hasError) {
      return (
        <div>
          <h2>Failed to load component</h2>
          <button onClick={() => window.location.reload()}>
            Retry
          </button>
        </div>
      );
    }
    return this.props.children;
  }
}

// Usage
<LazyLoadErrorBoundary>
  <Suspense fallback={<Loading />}>
    <LazyComponent />
  </Suspense>
</LazyLoadErrorBoundary>

Retry Logic:
------------
function retryImport(fn, retriesLeft = 3, interval = 1000) {
  return new Promise((resolve, reject) => {
    fn()
      .then(resolve)
      .catch((error) => {
        setTimeout(() => {
          if (retriesLeft === 1) {
            reject(error);
            return;
          }
          retryImport(fn, retriesLeft - 1, interval).then(resolve, reject);
        }, interval);
      });
  });
}

// Usage
const Dashboard = lazy(() => 
  retryImport(() => import('./Dashboard'))
);

9. LIMITATIONS
--------------
1. Server-Side Rendering (SSR)
   - React.lazy() doesn't work with SSR out of the box
   - Use libraries like @loadable/component for SSR
   
2. Named Exports
   - React.lazy() only supports default exports
   - Must wrap named exports in object
   
3. No Suspense for Data Fetching (yet)
   - Currently only for components
   - Full data Suspense is in development
   
4. Loading States
   - Can cause layout shifts if not handled
   - Need careful fallback design
   
5. Network Dependency
   - Requires network request for first load
   - Can fail on poor connections

10. REAL-WORLD USE CASES
------------------------
Use Case 1: Admin Dashboard
----------------------------
// Only admins need this heavy component
const AdminPanel = lazy(() => import('./AdminPanel'));

function App() {
  const { isAdmin } = useAuth();
  
  return (
    <div>
      {isAdmin && (
        <Suspense fallback={<AdminPanelSkeleton />}>
          <AdminPanel />
        </Suspense>
      )}
    </div>
  );
}

Use Case 2: Modal Dialogs
--------------------------
const SettingsModal = lazy(() => import('./SettingsModal'));

function Header() {
  const [showSettings, setShowSettings] = useState(false);
  
  return (
    <>
      <button onClick={() => setShowSettings(true)}>
        Settings
      </button>
      {showSettings && (
        <Suspense fallback={<ModalSkeleton />}>
          <SettingsModal onClose={() => setShowSettings(false)} />
        </Suspense>
      )}
    </>
  );
}

Use Case 3: Rich Text Editor
-----------------------------
// Heavy editor loaded only when needed
const RichTextEditor = lazy(() => import('./RichTextEditor'));

function CreatePost() {
  const [showEditor, setShowEditor] = useState(false);
  
  return (
    <div>
      {!showEditor ? (
        <button onClick={() => setShowEditor(true)}>
          Start Writing
        </button>
      ) : (
        <Suspense fallback={<EditorSkeleton />}>
          <RichTextEditor />
        </Suspense>
      )}
    </div>
  );
}

SUMMARY
-------
✅ Use React.lazy() for dynamic imports
✅ Wrap with Suspense to show loading states
✅ Split code at route and component levels
✅ Provide meaningful fallback UI
✅ Handle errors with Error Boundaries
✅ Measure and optimize bundle sizes
✅ Preload on hover for better UX
✅ Use for heavy or conditional components
✅ Avoid over-splitting small components
✅ Test loading states and error scenarios

Lazy loading and Suspense are powerful tools for improving application performance 
by splitting code and loading it on demand. Use them strategically to optimize 
initial load time and provide a better user experience.
