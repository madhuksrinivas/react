FORM HOOKS IN REACT
===================

Form hooks are specialized React hooks introduced in React 19 for handling form
submissions, actions, and optimistic updates. They simplify form state management
and provide better integration with server actions.

================================================================================

1. OVERVIEW
-----------
Form hooks provide declarative ways to handle form submissions:

useActionState:
- Manages form submission state
- Handles pending and result states
- Works with form actions

useFormStatus:
- Reads parent form's status
- Useful in child components
- Provides pending, data, method, action

useOptimistic:
- Shows immediate UI updates
- Reverts if action fails
- Great for better UX

Availability:
React 19+

For earlier versions, use:
- useState + useTransition
- Third-party libraries (Formik, React Hook Form)

2. USE ACTION STATE
-------------------
See UseActionStateHook.txt for detailed documentation.

Manages form actions with built-in pending states:

const [state, formAction, isPending] = useActionState(action, initialState);

Example:
--------
async function submitForm(prevState, formData) {
  const name = formData.get('name');
  
  if (!name) {
    return { error: 'Name required' };
  }
  
  await saveToServer(name);
  return { success: true };
}

function Form() {
  const [state, formAction, isPending] = useActionState(submitForm, {});
  
  return (
    <form action={formAction}>
      <input name="name" />
      <button disabled={isPending}>
        {isPending ? 'Submitting...' : 'Submit'}
      </button>
      {state.error && <p>{state.error}</p>}
    </form>
  );
}

3. USE FORM STATUS
------------------
Provides status of parent form (must be called inside form):

import { useFormStatus } from 'react';

function SubmitButton() {
  const { pending, data, method, action } = useFormStatus();
  
  return (
    <button disabled={pending}>
      {pending ? 'Submitting...' : 'Submit'}
    </button>
  );
}

function Form() {
  return (
    <form action={formAction}>
      <input name="email" />
      <SubmitButton />
    </form>
  );
}

Properties:
-----------
pending: Boolean - is form submitting?
data: FormData - form data being submitted
method: String - HTTP method (GET/POST)
action: String/Function - form action

Key Point: Must be called from component inside <form>

Use Cases:
----------
1. Reusable Submit Buttons
   function SubmitButton({ children }) {
     const { pending } = useFormStatus();
     return <button disabled={pending}>{pending ? 'Loading...' : children}</button>;
   }

2. Show Loading Anywhere in Form
   function FormStatus() {
     const { pending } = useFormStatus();
     return pending && <Spinner />;
   }

3. Conditional Fields
   function DynamicField() {
     const { data } = useFormStatus();
     const type = data?.get('type');
     return type === 'advanced' && <AdvancedOptions />;
   }

4. USE OPTIMISTIC
-----------------
Shows optimistic updates while waiting for async operations:

import { useOptimistic } from 'react';

function TodoList({ todos }) {
  const [optimisticTodos, addOptimisticTodo] = useOptimistic(
    todos,
    (state, newTodo) => [...state, newTodo]
  );
  
  async function addTodo(formData) {
    const todo = formData.get('todo');
    
    // Show optimistically
    addOptimisticTodo({ id: Date.now(), text: todo, pending: true });
    
    // Send to server
    await saveTodo(todo);
  }
  
  return (
    <>
      <ul>
        {optimisticTodos.map(todo => (
          <li key={todo.id} style={{ opacity: todo.pending ? 0.5 : 1 }}>
            {todo.text}
          </li>
        ))}
      </ul>
      <form action={addTodo}>
        <input name="todo" />
        <button>Add</button>
      </form>
    </>
  );
}

How it Works:
1. Call addOptimisticTodo with new value
2. UI updates immediately
3. Async operation runs in background
4. If success: state updates with real data
5. If failure: reverts to previous state

5. WHEN TO USE FORM HOOKS
--------------------------
Use Form Hooks When:
✅ Building forms with server actions
✅ Need pending states for submissions
✅ Want progressive enhancement
✅ Working with Next.js Server Actions
✅ Need optimistic UI updates
✅ React 19+ available

Use Traditional Approach When:
❌ React version < 19
❌ Complex validation logic needed
❌ Need libraries like Formik/React Hook Form
❌ Multiple forms on same page with shared state
❌ Need more control over submission flow

6. BEST PRACTICES
-----------------
1. Disable Inputs During Submission
   const { pending } = useFormStatus();
   <input disabled={pending} />

2. Provide Visual Feedback
   {pending ? 'Submitting...' : 'Submit'}

3. Handle Errors Gracefully
   {state.error && <ErrorMessage error={state.error} />}

4. Reset Forms on Success
   if (state.success) formRef.current.reset();

5. Use Optimistic Updates for Better UX
   Show immediate feedback for user actions

6. Validate Before Submission
   Check required fields in action function

7. Return Consistent State Shape
   return { error: null, success: true, data: {} };

8. Use FormData API
   Access form values: formData.get('fieldName')

9. Progressive Enhancement
   Forms work without JavaScript

10. Combine Hooks When Needed
    useActionState + useOptimistic for best UX

COMPARISON WITH ALTERNATIVES
-----------------------------
Form Hooks vs useState:
- Form Hooks: Built-in pending, works with forms, less code
- useState: More control, works anywhere, more boilerplate

Form Hooks vs Formik:
- Form Hooks: Native React, simpler for basic forms
- Formik: More features, better for complex validation

Form Hooks vs React Hook Form:
- Form Hooks: Integrated with React, good for server actions
- React Hook Form: Better performance, more flexible

MIGRATION EXAMPLE
-----------------
Before (Traditional):
---------------------
function Form() {
  const [data, setData] = useState({});
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    setLoading(true);
    setError(null);
    
    try {
      const formData = new FormData(e.target);
      await submitToServer(formData);
      setData({ success: true });
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input name="name" />
      <button disabled={loading}>
        {loading ? 'Submitting...' : 'Submit'}
      </button>
      {error && <p>{error}</p>}
    </form>
  );
}

After (Form Hooks):
-------------------
async function submitAction(prev, formData) {
  try {
    await submitToServer(formData);
    return { success: true };
  } catch (err) {
    return { error: err.message };
  }
}

function Form() {
  const [state, formAction, isPending] = useActionState(submitAction, {});
  
  return (
    <form action={formAction}>
      <input name="name" />
      <button disabled={isPending}>
        {isPending ? 'Submitting...' : 'Submit'}
      </button>
      {state.error && <p>{state.error}</p>}
    </form>
  );
}

Much simpler and more declarative!

SUMMARY
-------
✅ Form hooks simplify form handling in React 19+
✅ useActionState manages submission state
✅ useFormStatus provides form status to children
✅ useOptimistic enables optimistic UI updates
✅ Works great with server actions
✅ Reduces boilerplate code
✅ Provides better UX with built-in pending states
✅ Progressive enhancement support
✅ Not available in React < 19
✅ Use traditional methods or libraries for older versions

Form hooks make form handling more declarative and reduce the amount of
