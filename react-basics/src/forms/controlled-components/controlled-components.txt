CONTROLLED COMPONENTS IN FORMS
==============================

A controlled component is an input form element whose value is controlled by React state.
The React component that renders the form also controls what happens in that form on 
subsequent user input.

1. WHAT ARE CONTROLLED COMPONENTS?
-----------------------------------
In controlled components:
- React state is the "single source of truth"
- Form data is handled by React component
- Every state mutation has an associated handler function
- Input value is always driven by React state

Example:
--------
const [value, setValue] = useState('');

<input 
  type="text" 
  value={value}                    // Controlled by state
  onChange={(e) => setValue(e.target.value)}  // Updates state
/>

2. HOW CONTROLLED COMPONENTS WORK
----------------------------------
Flow:
1. User types in input
2. onChange event fires
3. Event handler updates state
4. Component re-renders
5. Input displays new state value

Key Point: The displayed value always matches the state value.

3. INPUT ELEMENTS
-----------------
Text Input:
-----------
const [name, setName] = useState('');

<input 
  type="text"
  value={name}
  onChange={(e) => setName(e.target.value)}
/>

Email Input:
------------
const [email, setEmail] = useState('');

<input 
  type="email"
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>

Password Input:
---------------
const [password, setPassword] = useState('');

<input 
  type="password"
  value={password}
  onChange={(e) => setPassword(e.target.value)}
/>

Number Input:
-------------
const [age, setAge] = useState(0);

<input 
  type="number"
  value={age}
  onChange={(e) => setAge(Number(e.target.value))}
/>

4. SELECT ELEMENTS
------------------
Single Select:
--------------
const [country, setCountry] = useState('');

<select value={country} onChange={(e) => setCountry(e.target.value)}>
  <option value="">Select</option>
  <option value="usa">USA</option>
  <option value="uk">UK</option>
</select>

Multiple Select:
----------------
const [colors, setColors] = useState([]);

<select 
  multiple 
  value={colors}
  onChange={(e) => {
    const values = Array.from(e.target.selectedOptions, option => option.value);
    setColors(values);
  }}
>
  <option value="red">Red</option>
  <option value="blue">Blue</option>
  <option value="green">Green</option>
</select>

5. CHECKBOX ELEMENTS
--------------------
Single Checkbox:
----------------
const [agreed, setAgreed] = useState(false);

<input 
  type="checkbox"
  checked={agreed}
  onChange={(e) => setAgreed(e.target.checked)}
/>

Multiple Checkboxes:
--------------------
const [hobbies, setHobbies] = useState([]);

const handleCheckbox = (hobby) => {
  if (hobbies.includes(hobby)) {
    setHobbies(hobbies.filter(h => h !== hobby));
  } else {
    setHobbies([...hobbies, hobby]);
  }
};

<input 
  type="checkbox"
  checked={hobbies.includes('reading')}
  onChange={() => handleCheckbox('reading')}
/>

6. RADIO BUTTONS
----------------
const [gender, setGender] = useState('');

<input 
  type="radio"
  name="gender"
  value="male"
  checked={gender === 'male'}
  onChange={(e) => setGender(e.target.value)}
/>

<input 
  type="radio"
  name="gender"
  value="female"
  checked={gender === 'female'}
  onChange={(e) => setGender(e.target.value)}
/>

7. TEXTAREA ELEMENTS
--------------------
const [comment, setComment] = useState('');

<textarea 
  value={comment}
  onChange={(e) => setComment(e.target.value)}
  rows={5}
/>

Note: Unlike HTML, React uses value prop for textarea, not children.

8. BENEFITS
-----------
✅ Single Source of Truth
   - State controls display value
   - No discrepancy between UI and data

✅ Easy Validation
   - Validate on every keystroke
   - Show real-time feedback

✅ Conditional Rendering
   - Enable/disable fields based on state
   - Show/hide based on values

✅ Easy to Reset
   - Clear form by resetting state
   - Simple form reset logic

✅ Predictable
   - Always know what's displayed
   - Easy to debug

9. BEST PRACTICES
-----------------
1. Use Single Handler for Multiple Inputs
   const [form, setForm] = useState({ name: '', email: '' });
   
   const handleChange = (e) => {
     const { name, value } = e.target;
     setForm({ ...form, [name]: value });
   };

2. Handle Different Input Types
   const handleChange = (e) => {
     const { name, value, type, checked } = e.target;
     setForm({
       ...form,
       [name]: type === 'checkbox' ? checked : value
     });
   };

3. Prevent Default on Submit
   const handleSubmit = (e) => {
     e.preventDefault();
     // Process form data
   };

4. Trim Values
   onChange={(e) => setName(e.target.value.trim())}

5. Convert Types When Needed
   onChange={(e) => setAge(Number(e.target.value))}

SUMMARY
-------
✅ Controlled components: React state controls input values
✅ Use value prop and onChange handler
✅ Single source of truth (React state)
✅ Easy validation and manipulation
✅ Predictable and debuggable
✅ Handle all input types consistently
✅ Use common handler for multiple inputs
✅ Always prevent default on form submit

Controlled components provide full control over form inputs, making them
the recommended approach for most React forms.
