REACT FORMS
===========

Forms are essential for collecting user input in web applications. React provides
flexible ways to handle form data through controlled and uncontrolled components.

1. CONTROLLED VS UNCONTROLLED COMPONENTS
-----------------------------------------
Controlled Components:
- React state controls the input value
- Value prop tied to state
- onChange handler updates state
- Single source of truth (React state)
- Real-time validation possible

Example:
const [name, setName] = useState('');
<input value={name} onChange={(e) => setName(e.target.value)} />

Uncontrolled Components:
- DOM maintains the input value
- Use refs to access values
- defaultValue prop for initial value
- Access value when needed (e.g., on submit)
- Simpler for basic forms

Example:
const nameRef = useRef(null);
<input ref={nameRef} defaultValue="" />
const value = nameRef.current.value;

When to Use:
✅ Controlled: Most cases, validation, dynamic forms
✅ Uncontrolled: Simple forms, file inputs, quick prototypes

2. FORM SUBMISSION
------------------
Basic Form Submit:
const handleSubmit = (e) => {
  e.preventDefault(); // Prevent page reload
  console.log(formData);
  // Process form data
};

<form onSubmit={handleSubmit}>
  <input ... />
  <button type="submit">Submit</button>
</form>

Important: Always call e.preventDefault() to prevent default form behavior

4. FORM VALIDATION
------------------
Client-side validation improves UX by providing immediate feedback.
See form-validation/ folder for complete examples.

Validation Timing:
- On blur: After user leaves field
- On change: While typing
- On submit: Before sending data
- Hybrid: Blur first, then change

5. INPUT HANDLING
-----------------
Techniques for processing user input:
- Transformation: uppercase, lowercase, capitalize
- Filtering: numbers only, letters only
- Masking: phone numbers, credit cards
- Debouncing: delay actions for search
- Trimming: remove whitespace

See input-handling/ folder for complete examples.

6. FORM LIBRARIES
-----------------
Popular form libraries:

Formik:
- Most popular
- Easy to use
- Built-in validation
- Good documentation

React Hook Form:
- High performance
- Minimal re-renders
- TypeScript support
- Small bundle size

Final Form:
- Framework agnostic
- Modular architecture
- Subscription-based updates

7. BEST PRACTICES
-----------------
✅ Use controlled components for most cases
✅ Always prevent default on submit
✅ Validate on both client and server
✅ Provide clear error messages
✅ Show errors after interaction (blur/submit)
✅ Disable submit when form is invalid
✅ Use proper input types (email, tel, number)
✅ Add labels for accessibility
✅ Use name attributes for all inputs
✅ Reset form after successful submit
✅ Handle loading states during submission
✅ Provide visual feedback for required fields

SUMMARY
-------
React forms can be handled in two main ways:
1. Controlled components (recommended) - React state manages values
2. Uncontrolled components - DOM manages values with refs

Choose controlled for most cases, especially when you need:
- Real-time validation
- Dynamic forms
- Conditional rendering
- Input transformation

Choose uncontrolled for:
- Simple forms
- File inputs
- Quick prototypes
- Performance-critical large forms

Always validate on both client and server, provide clear feedback,
and follow accessibility best practices.
