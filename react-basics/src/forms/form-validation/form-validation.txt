FORM VALIDATION IN REACT
=========================

Form validation ensures that user input meets specific requirements before submission.
React provides flexible ways to implement validation at different stages of user interaction.

1. TYPES OF VALIDATION
-----------------------
Required Fields:
- Check if field has value
- Most basic validation

Format Validation:
- Email format
- Phone number format
- URL format
- Date format

Length Validation:
- Minimum length
- Maximum length
- Exact length

Range Validation:
- Minimum value
- Maximum value
- Between range

Pattern Validation:
- Regular expressions
- Custom patterns

Cross-Field Validation:
- Password confirmation
- Date ranges
- Dependent fields

2. VALIDATION TIMING
--------------------
On Submit:
- Validate when form submitted
- Show all errors at once
- Pros: Less intrusive
- Cons: Late feedback

On Blur:
- Validate when field loses focus
- Pros: Good UX balance
- Cons: Delayed feedback

On Change:
- Validate on every keystroke
- Pros: Immediate feedback
- Cons: Can be annoying

Hybrid Approach (Recommended):
- On blur for first validation
- On change after first error
- Best user experience

3. CLIENT-SIDE VALIDATION PATTERNS
-----------------------------------
Pattern 1: Basic Validation
----------------------------
const [errors, setErrors] = useState({});

const validate = () => {
  const newErrors = {};
  
  if (!formData.email) {
    newErrors.email = 'Email is required';
  }
  
  if (!formData.password) {
    newErrors.password = 'Password is required';
  }
  
  return newErrors;
};

const handleSubmit = (e) => {
  e.preventDefault();
  const validationErrors = validate();
  
  if (Object.keys(validationErrors).length > 0) {
    setErrors(validationErrors);
    return;
  }
  
  // Submit form
};

Pattern 2: Field-Level Validation
----------------------------------
const validateField = (name, value) => {
  switch (name) {
    case 'email':
      return /\S+@\S+\.\S+/.test(value) ? '' : 'Invalid email';
    case 'password':
      return value.length >= 6 ? '' : 'Min 6 characters';
    default:
      return '';
  }
};

const handleChange = (e) => {
  const { name, value } = e.target;
  setFormData({ ...formData, [name]: value });
  
  const error = validateField(name, value);
  setErrors({ ...errors, [name]: error });
};

Pattern 3: Touched Fields
--------------------------
const [touched, setTouched] = useState({});

const handleBlur = (e) => {
  setTouched({ ...touched, [e.target.name]: true });
};

// Only show error if field was touched
{touched.email && errors.email && <span>{errors.email}</span>}

Pattern 4: Custom Validation Hook
----------------------------------
function useFormValidation(initialState, validate) {
  const [values, setValues] = useState(initialState);
  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});
  
  const handleChange = (e) => {
    const { name, value } = e.target;
    setValues({ ...values, [name]: value });
  };
  
  const handleBlur = (e) => {
    const { name } = e.target;
    setTouched({ ...touched, [name]: true });
    const error = validate(name, values[name]);
    setErrors({ ...errors, [name]: error });
  };
  
  return { values, errors, touched, handleChange, handleBlur };
}

4. ERROR HANDLING
-----------------
Display Errors:
---------------
Inline Errors:
{errors.email && <span style={{ color: 'red' }}>{errors.email}</span>}

Error Border:
<input 
  style={{ borderColor: errors.email ? 'red' : 'gray' }}
/>

Error Summary:
{Object.keys(errors).length > 0 && (
  <div>
    <p>Please fix the following errors:</p>
    <ul>
      {Object.values(errors).map((error, i) => (
        <li key={i}>{error}</li>
      ))}
    </ul>
  </div>
)}

Preventing Submission:
----------------------
const isValid = Object.keys(errors).length === 0;

<button type="submit" disabled={!isValid}>
  Submit
</button>

6. BEST PRACTICES
-----------------
1. Show Errors After Interaction
   - Don't show errors immediately
   - Wait for blur or submit

2. Provide Clear Error Messages
   ✅ "Email is required"
   ❌ "Invalid input"

3. Validate on Both Client and Server
   - Client: Better UX
   - Server: Security

4. Indicate Required Fields
   <label>Email *</label>
   or
   <label>Email <span aria-label="required">*</span></label>

5. Disable Submit Until Valid
   <button disabled={!isValid}>Submit</button>

6. Real-Time Validation for Long Forms
   - Show progress
   - Validate sections

7. Preserve User Input
   - Don't clear valid fields
   - Only clear on successful submit

8. Focus First Error Field
   if (firstErrorField) {
     firstErrorField.focus();
   }

7. COMMON VALIDATION RULES
---------------------------
Email:
/\S+@\S+\.\S+/.test(email)

Phone (US):
/^\d{10}$/.test(phone)
or
/^\(\d{3}\) \d{3}-\d{4}$/.test(phone)

URL:
/^https?:\/\/.+/.test(url)

Postal Code (US):
/^\d{5}(-\d{4})?$/.test(zip)

Credit Card:
/^\d{13,19}$/.test(card)

Password Strength:
- At least 8 characters
- Contains uppercase
- Contains lowercase
- Contains number
- Contains special character

/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$/

Date (YYYY-MM-DD):
/^\d{4}-\d{2}-\d{2}$/.test(date)

Username:
/^[a-zA-Z0-9_]{3,16}$/.test(username)

8. ACCESSIBILITY
----------------
1. Use aria-invalid
   <input aria-invalid={!!errors.email} />

2. Use aria-describedby
   <input aria-describedby="email-error" />
   <span id="email-error">{errors.email}</span>

3. Use aria-required
   <input aria-required="true" />

4. Focus Management
   - Focus first error on submit
   - Announce errors to screen readers

5. Error Announcement
   <div role="alert" aria-live="polite">
     {errors.email}
   </div>

Complete Example:
-----------------
function ValidatedForm() {
  const [form, setForm] = useState({ email: '', password: '' });
  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});
  
  const validate = (name, value) => {
    switch (name) {
      case 'email':
        if (!value) return 'Email is required';
        if (!/\S+@\S+\.\S+/.test(value)) return 'Invalid email';
        return '';
      case 'password':
        if (!value) return 'Password is required';
        if (value.length < 6) return 'Min 6 characters';
        return '';
      default:
        return '';
    }
  };
  
  const handleChange = (e) => {
    const { name, value } = e.target;
    setForm({ ...form, [name]: value });
    
    if (touched[name]) {
      const error = validate(name, value);
      setErrors({ ...errors, [name]: error });
    }
  };
  
  const handleBlur = (e) => {
    const { name, value } = e.target;
    setTouched({ ...touched, [name]: true });
    
    const error = validate(name, value);
    setErrors({ ...errors, [name]: error });
  };
  
  const handleSubmit = (e) => {
    e.preventDefault();
    
    const newErrors = {};
    Object.keys(form).forEach(key => {
      const error = validate(key, form[key]);
      if (error) newErrors[key] = error;
    });
    
    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      setTouched(Object.keys(form).reduce((acc, key) => ({
        ...acc, [key]: true
      }), {}));
      return;
    }
    
    console.log('Submit:', form);
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <input
          name="email"
          value={form.email}
          onChange={handleChange}
          onBlur={handleBlur}
          aria-invalid={!!errors.email}
          aria-describedby="email-error"
        />
        {touched.email && errors.email && (
          <span id="email-error" role="alert">
            {errors.email}
          </span>
        )}
      </div>
      
      <button type="submit">Submit</button>
    </form>
  );
}

SUMMARY
-------
✅ Validate on blur + onChange after error
✅ Show clear, specific error messages
✅ Use touched state to control error display
✅ Validate on both client and server
✅ Provide real-time feedback
✅ Use validation libraries for complex forms
✅ Make forms accessible (ARIA attributes)
✅ Disable submit when invalid
✅ Focus first error field
✅ Preserve valid user input

Good form validation improves user experience, reduces errors, and helps
users complete forms successfully on the first attempt.
