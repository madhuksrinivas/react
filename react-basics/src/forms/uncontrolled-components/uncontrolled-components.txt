UNCONTROLLED COMPONENTS IN FORMS
=================================

An uncontrolled component is an input form element that maintains its own internal state.
React doesn't control the value - the DOM itself maintains the state.

TABLE OF CONTENTS
-----------------
1. What are Uncontrolled Components?

================================================================================

1. WHAT ARE UNCONTROLLED COMPONENTS?
-------------------------------------
In uncontrolled components:
- DOM is the "single source of truth"
- Form data is handled by the DOM itself
- Use refs to get form values when needed
- Similar to traditional HTML forms

Example:
--------
const inputRef = useRef(null);

<input 
  type="text" 
  ref={inputRef}                   // Access via ref
  defaultValue="Initial"           // Set initial value
/>

// Get value when needed
const value = inputRef.current.value;

2. HOW UNCONTROLLED COMPONENTS WORK
------------------------------------
Flow:
1. User types in input
2. DOM updates the value automatically
3. React doesn't track changes
4. Access value via ref when needed (e.g., on submit)

Key Point: React doesn't control or know about the value until you read it.

3. USING REFS
-------------
Create Ref:
-----------
import { useRef } from 'react';

const inputRef = useRef(null);

Attach to Input:
----------------
<input type="text" ref={inputRef} />

Read Value:
-----------
const value = inputRef.current.value;

Set Value:
----------
inputRef.current.value = "new value";

Focus Input:
------------
inputRef.current.focus();

4. DEFAULT VALUES
-----------------
Use defaultValue (not value) for initial value:

Text Input:
-----------
<input type="text" ref={inputRef} defaultValue="John" />

Select:
-------
<select ref={selectRef} defaultValue="usa">
  <option value="usa">USA</option>
  <option value="uk">UK</option>
</select>

Checkbox:
---------
<input type="checkbox" ref={checkRef} defaultChecked={true} />

Textarea:
---------
<textarea ref={textareaRef} defaultValue="Initial text" />

Important: Use defaultValue/defaultChecked, not value/checked

5. FILE INPUTS
--------------
File inputs are always uncontrolled in React:

const fileRef = useRef(null);

<input type="file" ref={fileRef} />

// Get file on submit
const file = fileRef.current.files[0];

You cannot programmatically set file input values (security restriction).

6. WHEN TO USE UNCONTROLLED
----------------------------
✅ Use Uncontrolled When:
- Integrating with non-React code
- File inputs (always uncontrolled)
- Simple forms without validation
- Performance critical (large forms)
- Quick prototypes
- Legacy code migration

❌ Use Controlled When:
- Need real-time validation
- Conditional rendering based on input
- Multiple inputs affect each other
- Need to transform input (uppercase, format)
- Dynamic forms

7. CONTROLLED VS UNCONTROLLED
------------------------------
Controlled:
-----------
const [value, setValue] = useState('');

<input 
  value={value}
  onChange={(e) => setValue(e.target.value)}
/>

Pros:
- Real-time validation
- Instant UI updates
- Conditional logic
- Easy to test

Cons:
- More code
- Re-renders on every keystroke
- Performance impact on large forms

Uncontrolled:
-------------
const inputRef = useRef(null);

<input ref={inputRef} defaultValue="" />

Pros:
- Less code
- Better performance
- Simple forms
- Easy migration from HTML

Cons:
- No real-time validation
- Can't easily control value
- Harder to test
- Less React-like

8. BEST PRACTICES
-----------------
1. Access on Submit, Not During Typing
   ✅ const handleSubmit = () => {
        const value = inputRef.current.value;
      };
   
   ❌ Don't access ref in render function

2. Use defaultValue for Initial Value
   ✅ <input ref={ref} defaultValue="initial" />
   ❌ <input ref={ref} value="initial" />

3. Don't Mix Controlled and Uncontrolled
   ❌ <input ref={ref} value={state} />  // Don't do this
   
   Pick one approach per input

4. Reset Form Properly
   const handleReset = () => {
     inputRef.current.value = '';
   };

5. Handle Null Refs
   if (inputRef.current) {
     const value = inputRef.current.value;
   }

6. Use for File Inputs
   File inputs must be uncontrolled:
   <input type="file" ref={fileRef} />

Complete Example:
-----------------
function UncontrolledForm() {
  const nameRef = useRef(null);
  const emailRef = useRef(null);
  
  const handleSubmit = (e) => {
    e.preventDefault();
    
    const formData = {
      name: nameRef.current.value,
      email: emailRef.current.value
    };
    
    console.log(formData);
    
    // Reset form
    nameRef.current.value = '';
    emailRef.current.value = '';
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input type="text" ref={nameRef} defaultValue="" />
      <input type="email" ref={emailRef} defaultValue="" />
      <button type="submit">Submit</button>
    </form>
  );
}

Common Patterns:
----------------
1. Focus on Mount
   useEffect(() => {
     inputRef.current.focus();
   }, []);

2. Get Value on Submit
   const handleSubmit = (e) => {
     e.preventDefault();
     const value = inputRef.current.value;
   };

3. Clear Input
   const handleClear = () => {
     inputRef.current.value = '';
   };

4. Validate on Submit
   const handleSubmit = (e) => {
     e.preventDefault();
     if (!inputRef.current.value) {
       alert('Required field');
       return;
     }
   };

SUMMARY
-------
✅ Uncontrolled: DOM manages state, not React
✅ Use refs to access values
✅ Use defaultValue for initial values
✅ Good for simple forms and file inputs
✅ Better performance (no re-renders)
✅ Less code than controlled
✅ File inputs are always uncontrolled
✅ Access values when needed (on submit)
❌ No real-time validation
❌ Can't easily control input dynamically

Uncontrolled components are useful for simple forms and when integrating
with non-React code, but controlled components are generally recommended
for most React applications.
